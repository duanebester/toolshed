The Problem

SaaS was built for human buyers

AI agents are increasingly capable. But the moment an agent needs a tool it wasn't pre-configured with — fraud detection, compliance checking, geospatial analysis — everything stops. There's no way to find one, no way to pay for it, and no way to know if it's any good.

No Discovery

Tools are hardcoded or manually configured. There's no Yellow Pages for agent capabilities.

No Payment

No machine-native way to pay for a tool call. It's all API keys, billing dashboards, and enterprise contracts.

No Reputation

An agent can't know which provider is reliable, fast, or accurate without a human pre-vetting everything.

No Audit Trail

When an agent makes a decision based on a tool's output, there's no versioned, reproducible record of what happened.

Per-seat licenses, annual contracts, "schedule a demo" funnels — the entire commercial infrastructure of software was designed for humans. An agent that lives for 30 seconds can't sit through a sales call.

The Solution

npm + Stripe Connect for AI agents

ToolShed is a two-sided marketplace. Companies list tools with pricing, schemas, and SLAs. Agents discover and pay for them at runtime. Reputation is computed from verified usage — you can't fake it without spending real money.

How It Works

Agent needs fraud detection but doesn't have a fraud tool configured

Calls toolshed_search — finds tools ranked by quality, price, and latency

Calls toolshed_invoke — gateway handles payment, calls the provider, validates the response

Agent uses the result. Submits a proof-of-use review. Credits draw down automatically.

Competition

Competitive landscape

MCP Directories

Smithery, mcp.run, Glama

List tools for developers to browse and manually configure. No payment, no reputation, no way for an agent to discover tools on its own at runtime.

API Marketplaces

RapidAPI, AWS Marketplace

Built for human buyers — dashboards, sales cycles, per-seat pricing. None of that works when the buyer needs something in 200ms and disappears 30 seconds later.

Agent Frameworks

LangChain, CrewAI

Help you wire up tools you already have. They don't help your agent find, pay for, or trust tools it doesn't have.

Protocol Efforts

Anthropic MCP, Google A2A

Define how an agent talks to a tool. Protocol is just a field in our tool record. They're solving transport. We're solving commerce.

Our Insight

Everyone is fighting over how agents should talk to tools. That's plumbing. What doesn't exist is how agents find tools, know which ones are good, and pay for them — the commercial layer. MCP directories are building lists. API marketplaces are building storefronts for humans. We're building the package manager for an agent economy.
